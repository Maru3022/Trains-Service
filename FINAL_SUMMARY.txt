## 🔧 Как всё работает

### Outbox Pattern Flow
```
1. Client создает Train (POST /api/trains)
   ↓
2. TrainService.saveTrain() вызывается
   ├─ Сохраняет Train в БД
   └─ Вызывает TrainEventProducer.sendEvent()
   ↓
3. TrainEventProducer.sendEvent()
   └─ Сохраняет событие в таблицу outbox (PENDING статус)
   ↓
4. OutboxProcessor (каждые 5 сек)
   ├─ Находит все PENDING события
   ├─ Отправляет в Kafka
   └─ Обновляет статус (SENT или FAILED)
   ↓
5. Гарантированная доставка в Kafka ✅
```

## 🎓 Ключевые моменты

### Atomicity ✅
События сохраняются в БД в **одной транзакции** с данными:
```java
@Transactional
public Train saveTrain(Train train) {
    Train saved = trainRepository.save(train);        // DB
    trainEventProducer.sendEvent(createEvent(saved)); // Outbox DB
    return saved; // Commit both or nothing
}
```

### Reliability ✅
Асинхронный процессор повторяет отправку при ошибке:
```java
@Scheduled(fixedDelay = 5000) // Каждые 5 сек
public void processOutbox() {
    // Отправка с retry при ошибке
}
```

### Auditability ✅
История всех событий в БД:
```sql
SELECT * FROM outbox WHERE status = 'SENT' 
ORDER BY processed_at DESC;
```
